## Жизненный цикл сборки Maven

### Фаза clean

Команда mvn clean удаляет папку target со всеми предыдущими результатами сборки. 
В консоли видно как Maven удаляет директорию target, обеспечивая чистую сборку без артефактов от прошлых запусков.

### Фаза compile

Команда mvn compile компилирует исходный код Java в байт-код. 
В папке target создается поддиректория classes, содержащая скомпилированные .class файлы. 
Maven показывает процесс компиляции каждого класса.

В процессе подготовки к упаковке приложения Maven создает служебную папку maven-archiver, 
где хранятся метаданные, необходимые для формирования итогового архива.
При использовании различных генераторов кода может появляться директория generated-sources, содержащая автоматически сгенерированный исходный код.

Когда проект проходит фазу упаковки, в корне target появляется итоговый артефакт — файл с расширением jar, war или ear в зависимости от типа проекта. 
Для веб-приложений дополнительно создается поддиректория с развернутой структурой веб-приложения, готовой к деплою на сервер.

Папка maven-status — это служебная директория, которую Maven создает для внутреннего отслеживания состояния процесса сборки. 
Она содержит метаданные, которые использует плагин компилятора (чаще всего maven-compiler-plugin) для реализации инкрементальной компиляции.

Основное назначение этой папки — ускорение последующих сборок проекта. 
Когда вы повторно запускаете компиляцию, Maven обращается к файлам в maven-status, чтобы определить, 
какие исходные файлы были изменены с момента последней сборки. 
Внутри обычно находятся файлы с расширением .lst, которые хранят списки с информацией:

- Каких классах были скомпилированы в предыдущей сборке

- Зависимостях между классами - какие классы зависят от других

- Временных метках исходных файлов

- Пути к исходным кодам и скомпилированным классам

Благодаря этому механизму Maven перекомпилирует только те классы, которые действительно изменились,
или те, что зависят от измененных классов. 
Это значительно ускоряет процесс разработки, особенно в больших проектах.
Сами файлы в этой директории имеют технический характер и не предназначены для ручного редактирования — они автоматически управляются Maven.
### Фаза test

Команда mvn test выполняет модульные тесты проекта. 
Создается директория test-classes со скомпилированными тестами и surefire-reports с отчетами о выполнении тестов. 
В консоли отображается прогресс выполнения тестов и их результаты.

### Фаза package

Команда mvn package упаковывает скомпилированный код в готовый к распространению артефакт.
Для jar-проекта создается файл .jar в корне target.
Maven показывает процесс создания jar-архива, включая добавление манифеста и ресурсов.

Каждая фаза зависит от предыдущих - компиляция требует чистого состояния,
тестирование требует скомпилированного кода, а упаковка требует успешного прохождения тестов.
