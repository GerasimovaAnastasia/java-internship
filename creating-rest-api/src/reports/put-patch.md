## Разница между PUT и PATCH.

**PUT HTTP-метод** - используется для полной замены ресурса.

**PATCH HTTP-метод** - предназначен для частичного обновления.

В Spring Boot для работы с этими методами используются аннотации:
- `@PutMapping` - для обработки PUT-запросов
- `@PatchMapping` - для обработки PATCH-запросов

В случае с PUT-запросом важно передавать все поля сущности,
иначе вместо пропущенных полей будут null значения и старая информация пропадет. 
Метод перезаписывает все поля сущности.

В случае с PATCH-запросом можно (и подразумевается) замена только части данных сущности.
Например, пользователь сменил номер телефона, ему надо изменить только его, 
всю сущность переписывать избыточно. 
Данный метод не заменяет null значениями поля, неуказанные в теле запроса. 

В контексте задачи с книгами, метод patch может быть использован для изменения цены книги.
